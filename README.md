# file-sort

## 目的：设计一个统一win+andirod+iphone+hormony的文件统一管理的工具

## 对比：向日葵，联想超级互联、Flix、快牙、坚果云




| 功能模块 |                             描述                             | 技术考量                                                     |
| :------- | :----------------------------------------------------------: | ------------------------------------------------------------ |
| 跨端索引 |     建立全局文件索引库，不移动原始文件，仅建立索引链接。     | 需在各端开启辅助权限（如Android无障碍、iOS文件App Extension）。 |
| 高速互传 | 类似“快牙”的局域网直连，或基于WebRTC的P2P传输，绕过服务器带宽限制。 | 需解决NAT穿透问题，iOS后台运行限制。                         |
| 格式兼容 | 自动识别文件类型，对于不兼容格式（如手机上的PSD在Win上编辑）提供转换建议。 | 需集成轻量级格式转换引擎。                                   |
| 鸿蒙协同 | 利用鸿蒙的分布式能力，实现文件在设备间的无缝流转（流转而非拷贝）。 | 需接入鸿蒙SDK，申请相应权限。                                |

| 维度       | 向日葵 / 联想超级互联                                | Flix (微软)                                        | 快牙 / 坚果云                                    | File-Sort (本项目)                           |
| ---------- | ---------------------------------------------------- | -------------------------------------------------- | ------------------------------------------------ | -------------------------------------------- |
| 核心模式   | 远程控制/镜像为主，文件传输为辅。                    | 跨平台剪贴板/链接共享 (需微软账号生态)。           | 点对点传输 (快牙) 或 自动同步 (坚果云)。         | 统一索引 + 智能分拣 + 分布式管理。           |
| 跨平台能力 | 强 (Win/And/iOS)，但操作延迟较高。                   | 强 (仅限Win/And/iOS)，深度绑定Office/Windows生态。 | 快牙强(无视系统)，坚果云弱(主要同步特定文件夹)。 | 最强 (含HarmonyOS)，独立于特定大厂账号体系。 |
| 传输速度   | 依赖云端中转，速度受网络影响大。                     | 依赖网络，主要用于小文件/链接。                    | 快牙极快(热点直连)，坚果云依赖网速。             | 混合模式：局域网直连 + 云端索引。            |
| 文件管理   | 弱 (仅能看到对方设备画面进行操作)。                  | 极弱 (仅共享内容，不管理文件结构)。                | 坚果云强(网盘式管理)，快牙无管理(传完即走)。     | 强：提供统一的文件树视图和标签系统。         |
| 鸿蒙支持   | 一般 (作为普通Android设备处理，无法发挥分布式优势)。 | 无。                                               | 无。                                             | 原生支持 (利用分布式软总线，低延迟)。        |
